Security Tools v2.5

A Must-Use WordPress Plugin for Security Management and Administrative Control

Overview

Security Tools is a comprehensive Must-Use (MU) WordPress plugin designed to provide advanced security management, administrative control, and interface customization capabilities. The plugin operates invisibly, hidden from other administrators while maintaining full functionality.

This guide is written for WordPress administrators. You do not need to be a developer to use this plugin. Each feature is optional and can be enabled or disabled at any time.

Key Characteristics

  • Must-Use Plugin: Installed in wp-content/mu-plugins/, automatically activated.
  • Self-Hiding: Invisible to other administrators.
  • Modular Architecture: Clean, maintainable file structure.
  • Non-Destructive: All features are toggleable; nothing is permanently changed.

Who This Is For

This plugin is designed for site owners, agencies, and administrators who need extra control over WordPress behavior and the admin interface. It is especially useful for managing client sites or production environments where stability and security are critical.

Installation

As Must-Use Plugin (Recommended)

  1. Copy the entire security-tools folder to wp-content/mu-plugins/.
  2. Create a new file named security-tools-loader.php inside wp-content/mu-plugins/ with the following content: 
    <?php
/**
 * Plugin Name: Security Tools Loader
 * Description: Loads the Security Tools MU plugin.
 */

if ( file_exists( __DIR__ . '/security-tools/security-tools.php' ) ) {
    require_once __DIR__ . '/security-tools/security-tools.php';
}
  3. The plugin activates automatically.
  4. Access via Security Tools in the WordPress admin sidebar.

Quick Start

Get Up and Running in 5 Minutes

  1. Open Security Tools in the WordPress admin menu.
  2. Review General and decide whether to enable Autohide Menu.
  3. Visit System Controls and enable only the options you need.
  4. Use the Branding page if you want a custom login logo or message.
  5. Save settings on each page. Changes take effect immediately.

Using the Plugin

Navigation

The plugin appears as a top-level menu called Security Tools. Each feature is organized into its own tab. If Autohide Menu is enabled, you can still access all pages using the tab navigation at the top of the settings screen or by visiting this URL directly:

wp-admin/admin.php?page=security-tools

Saving Changes

Each settings page has its own Save Settings button. Changes on one page do not affect other pages. If you leave a page without saving, changes on that page are lost.

Tables and Checkboxes

Pages like Admins, Plugins, Themes, Widgets, Admin Bar, and Metaboxes use tables with checkboxes. Use the checkbox in the header to select or clear all items.

Feature Guides

General

Autohide Menu hides the Security Tools menu from the sidebar. The plugin stays accessible via the tab navigation and by URL. This is useful if you want fewer visible admin links for other users.

Reset All Settings clears every setting and returns the plugin to defaults. It does not uninstall the plugin or delete files.

Branding

Custom Legend displays a short message on the login screen and the admin footer. Leave it blank to disable.

Custom Login Logo replaces the WordPress logo on the login page. You can use JPG, PNG, GIF, WebP, or SVG.

Logo Link URL changes where the login logo links to. Leave it blank to link to your site homepage.

System Controls

These toggles change core WordPress behavior. Use them carefully:

  • Disable Updates: stops all update checks (core, plugins, themes).
  • Hide Admin Notices: removes dashboard alerts and update prompts.
  • Disable Emails: blocks all outgoing emails, including password resets.
  • Disable Email Verification: removes the email verification prompt.
  • Disable Plugin/Theme Controls: prevents installing, editing, or changing plugins/themes.
  • Disable Comments: removes comment functionality across the site.
  • Disable Frontend Admin Bar: hides the admin bar on the front-end.

Hide Login Page lets you set a custom login URL. If enabled, visiting wp-login.php returns a 404 for non-logged-in users. Always test the new URL in an incognito window and bookmark it.

Admins

Hides selected administrator accounts from the Users list. Hidden admins still have full access and can log in.

Plugins

Hides selected plugins from the Plugins list. This does not deactivate them.

Themes

Hides selected themes from the Themes list. The active theme remains active even if hidden.

Widgets

Hides dashboard widgets. Some plugins register widgets late; this tool preserves those hidden widgets even if they do not appear in the list.

Admin Bar

Hide admin bar items by selecting them from the list. Some items cannot be removed via WordPress APIs; for those, use Hide by CSS ID. Enter the element ID without a leading # (e.g., wp-admin-bar-example).

Metaboxes

Hides meta boxes on post/page editors. The plugin discovers metaboxes automatically when you edit content. If some metaboxes are missing, use the Scan for Metaboxes button to load all post types and build a complete list.

Wordfence Integration

If Wordfence Login Security is active, Security Tools hides the 2FA filter links on the Users page. This does not change Wordfence functionality.

FAQ

Does this plugin change my files?

No. It works by using WordPress hooks and settings. All changes are reversible.

If I hide a plugin or theme, is it disabled?

No. Hiding only removes it from the admin UI; it still runs as normal.

Can I recover if I lock myself out?

Yes. Rename or remove the security-tools folder in mu-plugins or disable the relevant option in the database.

Troubleshooting

I can't see the Security Tools menu

If Autohide Menu is enabled, access the settings directly:

wp-admin/admin.php?page=security-tools

I forgot my custom login URL

Disable the feature by renaming the plugin folder:

wp-content/mu-plugins/security-tools

Then log in at wp-login.php, restore the folder, and update the setting.

Metaboxes are missing from the list

Edit a post to trigger automatic discovery, or use the Scan for Metaboxes button on the Metaboxes page.

Operational Risks & Warnings

Warning: This plugin provides powerful tools that modify the core behavior of WordPress. Inappropriate use can lead to administrative lockout or security vulnerabilities.

1. Total Administrative Lockout

Combining Hide Login Page with Disable Emails creates a high risk of lockout. If you forget your custom login URL and cannot receive password reset emails, you will be unable to access your dashboard via standard means.

  • Recovery: You must access your server via FTP/SFTP and rename or delete the wp-content/mu-plugins/security-tools directory to disable the plugin and restore default WordPress behavior.

2. Security Vulnerabilities

The Disable Updates feature stops all core, theme, and plugin updates. While useful for stability:

  • Your site will not receive security patches.
  • Leaving this enabled indefinitely significantly increases the risk of your site being hacked.
  • Only use this if you have an alternative patch management strategy.

3. Hidden Site Components

Features that hide Administrators, Plugins, or Themes do not remove them; they continue to execute code. A hidden malicious plugin or compromised administrator account poses a severe threat that is harder to detect when hidden from the UI.
